---

### Product Backlog [p.3]

- 해야 할 모든 것의 우선순위에 따라 정렬된 목록
- 제품에 필요한 모든 기능
  - 프로젝트가 진행되면서, 개선 사항, 버그 수정 등도 포함
- Product Backlog Items (PBI)의 모음
  - ID + Title + User Story + Constrants

---

### Definition of Done (DoD) [p.4]

- 팀이 정의한 DoD 항목 목록
  - (예) 코드가 main branch에 merge됨
  - (예) 테스트 통과
  - (예) 문서화 완료
- 구현이 끝나고, 통과여부 확인

---

### Sprint Goal [p.5]

- Sprint Goal을 명확한 문장으로 작성
- Sprint Goal을 단순 작업 목록이 아닌 가치 목표로 이해
  - (좋은 예) 사용자가 계정을 생성하고 시스템에 최초로 접근할 수 있다
  - (나쁜 예) 로그인 기능 구현(X)

---

### Sprint Backlog [p.6]

- Product Backlog으로부터 Sprint Backlog으로 선택
- Product Backlog를 개발에 적합하도록 구체화시킴
- Sprint Backlog Item들로 구성
  - User Story 형식 (As a ~ I want ~ So that ~)

---

### Prompt 작성을 통한 구현 설명 [p.7]

- Prompt의 체계적 작성 과정
- Prompt의 버전관리
- Prompt의 구조화

Copyright 2026 SMU SELab., All rights reserved.

### Sprint Increment 설명 [p.8]

- 이번 Sprint에서 완성된 기능 설명
- "완료"의 기준을 기능 중심으로 설명
- 실행 동영상

---

### Sprint Review 결과 [p.9]

- 제품에 관한 사용자 / 팀 내부 피드백
- 변경 요청 사항
  - 새로 추가된 PBI
  - 우선순위 변경 사항

---

### Sprint Retrospective [p.10]

- 작업 방식에 대한 팀 내부 피드백
- 잘된 점 (What went well)
- 문제점 (What didn't go well)
- 개선 액션 (Action Items)

---

### 평가 기준 [p.11]

- 마감 날짜 준수
- 모든 항목의 적합성 및 표현의 명확성
- Prompt의 체계적 작성 수준
- 시연 영상 설명의 적절성
