# 📊 EduLex — WBS (Work Breakdown Structure)

> EduLex MVP Sprint 작업 분류 및 일정 계획서

---

## 📌 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | EduLex — 전공 어휘 학습 웹 플랫폼 |
| **Sprint 기간** | 1주 (병렬 개발) |
| **개발 방식** | 변형 스크럼 — 팀A / 팀B 병렬 개발 |
| **팀 구성** | 팀A (FE1+BE1) / 팀B (FE1+BE1) / 서기 1명 |
| **기술 스택** | React, Tailwind CSS, Supabase, OpenAI API |

---

## 🗓️ 전체 일정 개요

```
[분석·설계]  →  [Sprint 개발 (4단계 병렬)]  →  [통합·테스트]  →  [마감·제출]
```

| 단계 | 내용 | 담당 |
|------|------|:----:|
| **분석·설계** | 요구사항 정의, Backlog 작성, ERD 설계, Figma 프로토타입, API 명세 초안 | 전체 |
| **Sprint 1단계** | 인증 기반 + 캐릭터/전공 선택 | 팀A / 팀B |
| **Sprint 2단계** | 로그인 + 출석·공식단어장 | 팀A / 팀B |
| **Sprint 3단계** | 단어장 CRUD·PDF→AI + 나만의 단어장 | 팀A / 팀B |
| **Sprint 4단계** | 별가루 Realtime + 테스트·진행률 | 팀A / 팀B |
| **통합·테스트** | API 연동 검증, E2E 테스트, 보안 테스트 | 전체 |
| **마감** | 발표 자료(pptx), 시연 영상 촬영 | 서기 + 전체 |

## 📐 Definition of Done (DoD)

Sprint 내 각 SBI는 아래 기준을 모두 충족해야 **Done** 처리됩니다.

- [ ] 코드가 `develop` 브랜치에 merge됨
- [ ] FE·BE 결합 후 기능 정상 동작 확인
- [ ] 관련 API 연동 테스트 통과
- [ ] 코드 리뷰 완료 (팀 내 최소 1인 확인)
- [ ] GitHub Kanban 카드 **Done** 컬럼으로 이동
