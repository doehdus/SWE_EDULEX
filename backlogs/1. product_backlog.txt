[2. 메인 화면 (Main / Home)]
----------------------------------------------------------------

[UPD][H01] 캐릭터 프리뷰 + 전공 선택 + 칭호 선택
- User Story: 사용자는 메인 화면에서 자신의 캐릭터와 현재 상태(레벨, 닉네임, 책갈피)를 확인하고, 캐릭터 하단에서 전공(최대 2개)과 칭호를 직접 선택할 수 있다.
- Constraints: 캐릭터 이미지, Lv·닉네임·책갈피·현재 선택 전공·칭호 표시, 상점 이동 버튼 포함
              캐릭터 하단 전공 선택 UI(최대 2개), 칭호 선택 UI(보유 칭호 중 1개)
              전공 변경 시 N01 공식 단어장 조회 자동 갱신 연동
              칭호 시스템(칭호 종류·획득 조건 테이블)은 Post-MVP에서 확장 예정
              MVP에서는 active_title 필드만 유지, 칭호 선택 UI는 보유 칭호 없으면 비활성화
- 우선순위: Medium
- ※ v1.2: 전공 선택·칭호 선택 기능 추가 (L01 전공 선택 제거에 따른 이동)
- ※ v1.4: 칭호 시스템 Post-MVP 확장 예정 명시

[UPD][H02] 출석 체크 / 스트릭
- User Story: 사용자는 메인 화면에서 간편하게 출석을 확인하고 보상(책갈피)을 받을 수 있다.
- Constraints: 스트릭 시각화(풀이 Lv에 따라 진하기 표현), 하루 1회 체크 가능
              책갈피 지급 시 star_dust_logs에 감사 로그 기록 (reason: 'attendance')
              책갈피 잔액 업데이트와 로그 기록은 트랜잭션으로 묶어 처리
- 우선순위: Medium
- ※ v1.4: 책갈피 보상 감사 로그 Constraints 추가

[NEW][H03] 최근 단어장 / 최근 테스트 요약
- User Story: 사용자는 메인 화면에서 최근에 이용한 단어장과 테스트 기록을 빠르게 확인할 수 있다.
- Constraints: 최근 단어장(user_wordbooks 최근 수정 기준 1건) 및 최근 퀴즈 결과(quiz_results 최근 1건) 요약 표시
              해당 단어장(/wordbook/my) 및 테스트 페이지(/quiz)로 이동 가능
              항목 없을 때 빈 상태 처리
- 우선순위: Medium
- ※ v1.7: 신규 추가

[UPD][H04] PDF → AI 단어장 생성 바
- User Story: 사용자는 메인 화면의 입력 바에서 PDF를 업로드하여 AI가 자동 생성한 단어장을 바로 만들 수 있다.
- Constraints: PDF 파일 업로드, 3단계 파이프라인으로 단어 추출 및 단어장 생성
              1단계: 순수 JS PDF 텍스트 파서 (BT/ET 블록 파싱) — 로컬 처리, 텍스트 레이어 있는 PDF만 지원
              2단계: TF-IDF 핵심 단어 추출 (상위 50개 선별) — 로컬 처리, 외부 API 없음
              3단계: Groq API (llama-3.3-70b-versatile) 연동 — 선별 단어 목록 전달, 최대 30단어 단어장 JSON 생성
              단어장 2개 초과 시 업로드 차단 (API 레벨 + DB 트리거 이중 방어)
              Supabase Edge Function 배포 (--no-verify-jwt), 내부에서 Auth REST API로 사용자 검증
              스캔 이미지 PDF 미지원 — 텍스트 추출 실패 시 명시적 에러 반환
- 우선순위: High
- ※ v1.2: OpenAI API로 명시
- ※ v1.5: Gemini 2.0 Flash로 변경, PDF 처리 방식을 multimodal inline_data로 전환
- ※ v1.6: Gemini API 키 소진으로 전면 재설계 — 순수 JS 텍스트 파서 + TF-IDF + Groq API로 전환

[---][H05] 책갈피(재화) 실시간 조회
- User Story: 사용자는 메인 화면에서 보유 중인 특수 재화(책갈피) 수량을 실시간으로 확인할 수 있다.
- Constraints: Supabase DB와 실시간 동기화
- 우선순위: High

[---][H06] 상점 이동
- User Story: 사용자는 획득한 포인트(책갈피)로 아이템을 구매할 수 있는 상점 페이지로 이동할 수 있다.
- Constraints: 캐릭터 프리뷰 영역 내 상점 버튼으로 접근
- 우선순위: Medium

[---][H07] 랭킹 요약
- User Story: 사용자는 메인 화면에서 현재 자신의 랭킹 순위나 상위권 유저 정보를 빠르게 확인할 수 있다.
- Constraints: 보유 책갈피 수 기준 랭킹, 상세 랭킹 페이지로 이동 가능
- 우선순위: Low

[---][H08] 칭호 표시
- User Story: 사용자는 획득한 칭호를 캐릭터 옆에 표시하여 자신의 성취를 확인할 수 있다.
- Constraints: 보유 칭호 중 1개 선택 표시
- 우선순위: Low
