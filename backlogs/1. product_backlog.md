## 2. 메인 화면

### `[UPD]` H01 · 캐릭터 프리뷰 + 전공 선택 + 칭호 선택 🟡 Medium

> 사용자는 메인 화면에서 자신의 캐릭터와 현재 상태를 확인하고, 캐릭터 하단에서 전공(최대 2개)과 칭호를 직접 선택할 수 있다.

- 캐릭터 이미지, Lv·닉네임·책갈피·현재 선택 전공·칭호 표시, 상점 이동 버튼 포함
- 캐릭터 하단 전공 선택 UI(최대 2개), 칭호 선택 UI(보유 칭호 중 1개)
- 전공 변경 시 N01 공식 단어장 조회 자동 갱신 연동
- 칭호 시스템(칭호 종류·획득 조건 테이블)은 Post-MVP에서 확장 예정
- MVP에서는 active_title 필드만 유지, 칭호 선택 UI는 보유 칭호 없으면 비활성화
- ※ v1.2: 전공 선택·칭호 선택 기능 추가 / ※ v1.4: 칭호 시스템 Post-MVP 확장 예정 명시

### `[UPD]` H04 · PDF → AI 단어장 생성 바 🔴 High

> 사용자는 메인 화면의 입력 바에서 PDF를 업로드하여 AI가 자동 생성한 단어장을 바로 만들 수 있다.

- PDF 파일 업로드, **Gemini 2.0 Flash API (multimodal)** 연동하여 단어 추출 및 단어장 생성
- PDF를 base64로 인코딩하여 Gemini inline_data로 직접 전달 (텍스트 추출 단계 없음)
- 단어장 2개 초과 시 업로드 차단 (API 레벨 + DB 트리거 이중 방어)
- Supabase Edge Function 배포 (`--no-verify-jwt`), 내부에서 Auth REST API로 사용자 검증
- ※ v1.2: OpenAI API로 명시 / ※ v1.5: Gemini 2.0 Flash로 변경, multimodal inline_data 방식으로 전환
